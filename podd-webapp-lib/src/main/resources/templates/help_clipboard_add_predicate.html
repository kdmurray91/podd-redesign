Cloning Predicates via the Browser Pane


Predicates

Predicates determine the linkage between a parent object (the subject) and its child object (the object). The predicate is explicit in the PODD domain model and therefore clearly stated in the Browser Pane of a project hierarchy.

Consequently predicates can be considered as groupings of objects and by selecting a predicate for the copy operation, the user can effectively copy all children objects associated with the predicate (shallow paste) or an entire hierarchy of objects associated with the predicate (deep paste). When pasting a predicate there is no requirement to select the relationship, as that is already explicit.


Copy Predicates

The user selects the Browse Project link of the project to be copied from and expands it to find the predicate that the user wants to copy. See figure 16.1.1.

Figure 16.1.1 – Locate the predicate to be copied in the Browser Pane.

The user clicks on the Copy link next to the predicate. A message (an alert) will say Predicate copied to the PODD Clipboard. The user can repeat this action to add more predicates (and objects) to the Clipboard.


Paste

After copying all the predicates to the Clipboard, the user then selects the Browser view of the project to be pasted to and clicks on the Paste link next to the object which they wish to paste the predicate to.

The user is then presented with the Clipboard view (See figure 16.1.2). The Clipboard is opened in a new tab.

Figure 16.1.2 – Selecting the predicate to paste from the Clipboard

As shown in Figure 16.1.2, the following features are displayed on the Copy Clipboard:

  • At the top of the Clipboard is a section describing the object or predicate to be pasted to, i.e., the Paste target’s title and object type.
  • The Clipboard provides a list of all predicates and objects copied, however only predicates and objects that can be pasted according to the domain model will display a checkbox that will allow them to be pasted to the Paste Target.
  • As per figure 16.1.2, for each item displayed in the Clipboard the following information will be presented if available:
    • A selection box to select the object/predicate for pasting. Multiple objects/predicates can be selected at once.
    • An icon representing the PODD type (object, predicate, file) as displayed in the browser.
    • The Title of the object/predicate in the Clipboard. If it is a predicate then the predicate type is displayed followed by the number of direct children objects of this predicate.
    • The Type of the object. Type will not be displayed for predicates on the Clipboard.
    • The Parent of the object/predicate, meaning the title of the parent object.
    • The Parent Type or object type of the parent object.
    • A Depth drop down box. Options are Shallow (default) or Deep. Depth drop down boxes will not be displayed for objects or predicates that cannot be pasted, or for file stream objects since the concept of depth do not apply to these objects.
    • A Relationship drop down box, which displays the predicates available for joining the copied object to its new parent (the Paste Target). Relationships will not be displayed for predicates on the Clipboard.

The user selects the predicates or objects to be pasted to the paste target, the user chooses between a Shallow and Deep copy and chooses the Relationship or predicate to link the paste target to the Clipboard object (if available).

The user clicks Paste (Note there is a Paste button at the top and bottom of the list, for convenience). The system will paste the objects, meaning it will clone the objects copied from and link them to the parent object with the specified predicate.

The Clipboard screen will be busy until the paste operation is complete. When the paste operation is complete, the user will be directed to a Copy & Paste Results screen (see figure 16.1.3).

Figure 16.1.3 – The Copy & Paste Results screen

This Copy and Paste screen shows the following information:

  • A summary of the parent object. Note that if a predicate was pasted to, then it is the parent object displayed here and not the predicate.
  • A list table of the objects that were in the paste operation and the status of the paste operation.
  • The hierarchy of the target parent object, expanded to show the immediate children so that the user may verify the paste was correct. Objects titles in the hierarchy are hyperlinked.
