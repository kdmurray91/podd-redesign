Copy and Paste


Overview

PODD objects, whether data or metadata objects, are linked to a project through the project hierarchy. A feature of PODD to support data sharing is the ability to copy PODD objects from one place and paste it to another. This can be from one project to another, or within a single project. Copy, Paste and Clone in this section are difined as follows:

  • Copy: means to copy objects, predicates and file streams to the clipboard.
  • Paste: means to paste objects, predicates and file streams to their new location.
  • Clone: making a copy of an object or set of objects through a copy and paste function.

Actors

  • Public Access users do not have the ability to copy, paste or view the clipboard.
  • Registered users can copy any objects/ they can see. They can paste objects/predicate as determined by their project role. They can edit their Clipboard.
  • Project Creator: as per registered users.
  • Administrators can copy objects from all projects and paste to all projects. They can edit their Clipboard.

For users with project specific roles:

  • Project Administrators can copy objects to and from projects they have admin access to.
  • Project Members can copy objects to and from projects they have member access to.
  • Project Observers can copy objects from a project that they have observer access to.

Rules and Restraints

Copy and Paste can be done via Object View or Browser Pane. Either way,  the following rules and constraints apply to Copy and Paste:

  1. Objects can be copied by Predicate or by Object.
  2. A Project object cannot be copied to the Clipboard.
  3. A Project object can be pasted to.
  4. An object or predicate can only be pasted to places that are allowable by the domain model. E.g. a Project Plan object cannot be pasted to an Investigation object, but can be pasted to a Project object.
  5. An object can be cloned between projects, as defined by the user's project roles.
  6. An object can be cloned within projects, as defined by the user's project role.
  7. An object can be cloned to itself (e.g. the user may wish to create a hierarchy of materials by cloning one object to different levels and then going back to modify the details).
  8. When pasting individual or hierarchies of predicates/objects the references will not be maintained, unless a deep paste is performed on a predicate/object and the references in the hierarchy so pasted refer to objects contained within the hierarchy. e.g. If a has Investigation predicate is deep pasted and it has Data objects that reference Material objects within the same hierarchy, then the integrity of these references need to be maintained.
  9. Pastes are performed in real time, the users screen will lock while a paste is performed.
  10. The clipboard has a limit of 40 items (This value is a system parameter, it can be updated). It works on a FIFO basis. If the clipboard is full the user is advised that they are copying to a full clipboard each time they copy.
  11. The on-screen rendering of the Clipboard shows all objects in the Clipboard at all times (i.e. no pagination).
  12. The Clipboard lists its contents in the order they were copied, from most recent to least recent (i.e. no sorting).
  13. Data streams representing files can be copied to the clipboard, they should be called File Streams as the term 'Data Streams' is confused with Data type objects.
  14. Multiple objects and predicates can be pasted at once, providing the model supports it.

Objects can be cloned via Browser Pane or Obeject view. Predicates can only be cloned via Browser Pane. They will be described in details as follows:

  • Cloning Predicates via the Browser Pane
  • Cloning Objects via the Browser Pane or the Object View

Also, objects copied to the Clipboard can be edited (see Editing the Clipboard) and objects deleted between copy and paste can be handled accordingly (see Handling deleted Objects).
