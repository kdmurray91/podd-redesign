Adding a Child Object


With the exception of a Project, all concrete PODD objects are instantiated with a reference to the parent object. However, not all objects can be a parent of another object. For example, a Project Plan can not have a child object.

Authorised users can add a child object by clicking the Add Child Object button when viewing the parent object. If the data model does not allow an object to have child objects, the Add Child Object button won't be available.


Select Child Object Type (and Parent-Child Relationship)

When the Add Child Object button is clicked, the Add Child Dialog is displayed as shown in Figure 14.1.1 below.

Figure 14.1.1 – Screenshot of Add Child Dialog

The Dialog contains a drop down list containing all "Child Object Type - Parent-Child Relationship" combinations possible for the parent object as shown in Figure 14.1.2 below.

Figure 14.1.2 – Screenshot showing possible "Child Object Type - Parent-Child Relationship" combinations

When satisfied with the child object type and relationship, the user clicks Continue. (To cancel the process, click the Dialog's close button to return to the previous View Object page).

The user is presented with an updated Add Child Object screen. The screen is different depending on the type of the child object.

Figure 14.1.3 – Example creenshot for Add New Platform


Fill in the Child Object details

The Child Object fields appearing are determined by the type of Child Object. The user has to fill these in and click the Submit button when satisfied. Validation is performed and if any fields are in error then the user is returned to the top of the page. An error box will appear at the top of the page describing the nature of the errors.

On successful submission, the following universal attributes will be automatically recorded against the new child object:

  • URI is determined by PODD.
  • Creator is set to the current user.
  • Creation Date is set to the current date.

The system confirms the submission to the administrator by switching to the View tab and displaying the object details page in view (read only mode). See: Viewing a Child Object for details.

The user can also abort the process and return to view the parent object by selecting the Cancel button.
