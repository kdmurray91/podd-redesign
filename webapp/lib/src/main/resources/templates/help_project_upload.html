Project Upload

For authorised users, i.e., Project Creators and Repository Administrators, PODD provides the option of uploading a new Project in the form of an RDF file. For this purpose, there is an Upload Project as RDF button listed on the Listing Projects screen. Once the button is clicked, the Upload new Project page is displayed.

The Upload new Project page requires the user to select a file containing the project details and upload it as shown in Figure 10.5.1.

 Figure 10.5.1 – Screenshot of the Upload new Project page

Once the Upload button is clicked, the RDF file is uploaded to the PODD server, validated and a new Project created.

If the Project is successfully created, the following screen is displayed.

 Figure 10.5.2 – Screenshot after successful File Upload

A Project is stored as an Ontology inside PODD. The screen displays information about this ontology:

  • Ontology IRI: A URI which identifies this Project
  • Version IRI: A URI which identifies the current version of the Project
  • Inferred IRI: A URI which identifies inferred data about the current version of the Project

It is not necessary for users of the PODD HTML interface to know about these URIs.
