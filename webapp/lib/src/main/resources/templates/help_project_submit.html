Submit or Cancel


At the bottom of the Create Project Page, there are two buttons displayed (Figure 10.1)

  • Submit
  • Cancel

The user can select one of the buttons based on the job requirement .


Submit

The user selects Submit. All the information entered is submitted to the system. Validation is performed.

If any fields are in error then the user is returned to the top of the page. An error box will appear at top of the page describing the nature of the errors and remedial action.

If no error is detected, the system confirms the submission to the user by switching to the view tab and displaying the project details page in View Project mode (See: Viewing a Project for details).

The following additional values will be automatically recorded against the Project concrete object:

  • Project URI, which is determined by PODD,
  • Creator is set to the current user,
  • Creation Date is set to the current date,
  • Project Status is set to 'Active',
  • Publication Status is set to 'Not Published',
  • Last modified by is set to the current user,
  • Last modification date is set to the current date.

Please note that where a project is created by a Project Creator and they have removed themselves from the Project Participant fields, they will not be able to view the project details later. In this circumstance they are not directed to the View Project page. A screen advising them they have insufficient access to the project will be displayed.


Cancel

If the user clicks the Cancel button, the Create Project page is abandoned. All information entered will be lost. New Project is not created and the user is then returned to the List Project Page.
