Deleting a Child Object


Overview

When an object is deleted, it is deleted in the underlying repository, and cannot be recovered.

Deleting an object is cascaded to any child objects that it may have. Therefore, it is important to carefully consider the effects of deleting an object.


Actors

  • Repository Administrators, Project Administrators and Project Members can delete child objects.

Delete an Object

On the View Object page, if the user has the appropriate authority the object view will display a Delete button.

When the user clicks the Delete button, a Confirmation Dialog is presented. The Dialog contains the object's Title and asks the user to confirm that they wish to delete this object. If the object has child objects, the displayed message advises that these too will be deleted, as shown in Figure 15.1.1.

Figure 15.1.1 – Example screenshot of delete confirmation dialog for an object with Child Objects.

The user clicks the Confirm button. The system deletes the object and any child objects it may have. If the user does not wish to proceed with the delete, press the Cancel button. The screen is returned to the Object View.

After successful deletion, the user is directed to the View Object screen for the parent object where it can be observed that the deleted object is no longer available as a child.
