Adding Project Participants


Project Participants are users who have Project specific roles. They have nominated CRUD accesses to the project while the project has a Project Status of current.

When a Project is created, by default the creator is given the Principal Investigator and Project Administrator roles.

For authorised users, i.e. Project Administrators, there is an Edit Participants button in the View Project screen. When the button is clicked, the Project Participants screen is displayed.

Figure 10.2.1 – Screenshot of the Project Participants page

As shown in Figure 10.2.1, there are four types of project participants (see also Project Specific Roles). They are:

  • Principal Investigator *
    • This field is mandatory, a project must have exactly one Principal Investigator at any time.
    • A Principal Investigator should also be a Project Administrator in order to administer the Project.
    • Principal Investigator can be replaced.
    • To edit Principal Investigator, hover the mouse pointer over the current entry. The label will change into an editable text box.
    • Project Administrator
      • The project can have multiple Project Administrators.
      • A default Project Administrator is already inserted in this field, being the person creating the project. This default can be edited out of the text field. Note this will have consequences when the creator submits the project.
    • Project Members
      • Project Members have Create, Read and Update access to all Project objects, and Delete access to internal Project objects.
    • Project Observers
      • Project Observers have Read only access to all Project objects.

    Adding project participants makes use of the Auto-Completion functionality.

    The user adds participants to each field as required. Please be advised that only existing registered users can be added in the fields. One user can have several roles in a project.
