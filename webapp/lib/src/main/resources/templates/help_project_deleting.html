Deleting a Project


Overview

When a Project is deleted, the Project and all its child objects are deleted in the underlying repository, and cannot be recovered. Therefore, it is important to carefully consider the effects of deleting a Project.

Repository Administrators and Project Administrators can delete Projects.


Delete a Project

On the View Project page, if the user has the appropriate authority, the page will contain a Delete Project button.

When the user clicks the Delete Project button, a Confirmation Dialog is presented. The Dialog contains the Project's Title and asks the user to confirm that they wish to delete this Project. If the Project has child objects, the displayed message advises that these too will be deleted, as shown in the Figure below.

Figure – Example screenshot of delete confirmation dialog for a Project with Child Objects.

The user clicks the Confirm button. The system deletes the Project and any child objects it may have. If the user does not wish to proceed with the delete, press the Cancel button. The screen is returned to the Project View.

After successful deletion, the user is directed to the List Projects screen where it can be observed that the deleted Project is no longer listed.
