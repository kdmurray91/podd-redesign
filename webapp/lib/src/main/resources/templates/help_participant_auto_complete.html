Auto-completion Functionality


User Auto-completion is a very useful function in PODD. The system lists all users that match the characters typed in the field. The user can then select the Project Participant from the appearing list without typing the complete user name. 

Figure 10.2.1.1 shows an example of the functionality. When "re" is typed in the Project Member field, a drop down list which displays all users with letters "re" in their user name, first name or last name pops up. The search is not case-sensitive. The user can select the desired user from this list.

 

Figure 10.2.1.1 – Detail showing selectable list to autocomplete fields

The detailed process of the auto-completion functionality is described as follows:

  1. The user starts to type in the required field.
  2. After at least 2 characters are typed, the system will list all available users that:
    • Have a first name that contains the character combination,
    • Have a last name that contains the character combination, or
    • Have a username that contains the character combination.
  3. As the user types in subsequent letters, numbers or symbols the list will be further filtered.
  4. The list will be in the form of a drop down box below the required text field.
  5. Each entry in the list contains the full name and institution of the user.
  6. The user can use the up and down arrow keys to select the desired project participant.
  7. The user hits enter once the selection is made (or clicks on a selection), and the text field is updated with the full name and institution of the participant.
  8. The user clicks the Add button or leaves the text field to confirm adding the participant.
  9. The text field is cleared and the PODD server updated in the background.
