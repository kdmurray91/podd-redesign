Listing Projects


Overview

A Project Object is considered the foundation object for any projects in PODD. That is, objects will exist in a hierarchy of relationships with a Project object as the top node. For this reason the first view that a user will see is the Projects tab with a list of projects that he/she may access.


Actors

All users, whether registered or public access, will be able to list projects. Two types of projects are defined in the system.  They are:

  • Published Projects: Public access projects.
  • Unpublished Projects: Projects that are not published. Non-admin users will only see projects that they are associated with. *

Different types of projects will be listed based on the user's repository roles:

  • Public users can only list Published Projects.
  • Registered users/Project Creators can list (their) Unpublished Projects and Published Projects.
  • Administrators can list All Projects in the PODD instance.

* A user is associated with a project if, they are defined as either a Project Administrator/Principal Investigator (CRUD access), Project Member (CRU) or Project Observer (R) of the project.


The Project Pane

When a user navigates to the PODD website without logging in, the default view is the Projects Pane. All Public access (i.e. Published) projects are listed in the Project Pane.

Once the user logs in, the Project Pane will list all unpublished projects the user is associated with and all public access projects (Published Projects).

 

Figure 8.1 – Default Listing Projects View in Projects Tab for a registered user

As shown in Figure 8.1, for each Project object listed in the Project Pane the following information is available:

  • The Project Title (Links to Browser View of the project);
  • Project description. First 200 characters, whole words (no breaks in words) or to first line return, which ever comes first;
  • A Browse Project Link, which takes the user to the Browser View of the project (same as the link behind the project title).

Unpublished Projects and Published Projects are separated in the view.

A Repository Administrator will, by default, have a listing of All Projects.

Both Unpublished Projects and Published Projects can be hidden by clicking the collapse icon next to the headings (see Figure 8.1).


Action Buttons

There are also additional buttons displayed for different user roles:

  • Create Projects button is only visible to Project Creator and Repository Administrator. It will take user to the Create Project function (see Creating a Project).  
  • Upload Project as RDF button is visible to Project Creator and Repository Administrator. It will take user to the Upload New Project function.
