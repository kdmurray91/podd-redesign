Creating a Project


Overview

PODD organizes all data into projects. To record any data or metadata, the first thing is to create a concept project object.


Actors

  • Public users and registered users can’t create projects.
  • Project Creators may create projects. They will not have any further access over the project unless they define themselves with a project role during project creation. Note the default behaviour of the system is to give the Project Creator Project Administrator and Project Principal Investigator roles.
  • Administrators may create and modify projects.

Creating a Project

For the authorised users, i.e., the Project Creator and Repository Administrator, there is a Create Project button listed on the Listing Projects screen. Once the button is hit, the Add new Project page is displayed. 

Adding a new Project consists of the following two steps:

  • add project details
  • add project participants

At the bottom of the page, two buttons are listed for user to take some actions. They are: Submit and Cancel.
