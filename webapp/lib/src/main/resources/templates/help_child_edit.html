Editing a Child Object


The Edit Object screen is arrived at by clicking the Edit button in the View Object screen.

The user is presented with an Edit Object screen (See figure 14.3.1). The screen is similar to the second stage of the Add Child Object screen, with the text boxes filled. All the sections are displayed in an expanded state by default.

Figure 14.3.1 – Example screenshot for editing an Investigation object


Object Details

The Object Details section displays all the attributes of the object. Figure 14.3.1 shows an example of an Investigation object being edited. Object Title and Description can be modified. Some other fields may be available for editing depending on the type of the object.

If the object allows RefersTo links, these too are displayed under the Object Details section. RefersTo fields should be modified using the auto completion functionality as shown in Figure 14.3.2.

Figure 14.3.2 – Example screenshot for using auto completion on a RefersTo field


Action Buttons

When satisfied with the changes, the user clicks Submit.

Validation is performed, and if any fields are in error then the user is returned to the top of the page. An error box will appear at top of the page describing the nature of the errors and remedial actions.

The system confirms the submission to the administrator by switching to the view tab and displaying the object details page in view (read only mode).

If the user clicks the Cancel button, the Edit Object screen will be abandoned.  The screen is returned to the View Object page without any change.
