Viewing a Project


Overview

Users can view projects as determined by their repository role or project specific roles (same as Listing Projects): 

  • Public users can view public access projects.
  • Registered users can view public access projects and the projects they are associated with.
  • Project Creators view public access projects and the projects they are associated with. They will not have any view access over the project they created unless they define themselves with a project role during project creation.
  • Administrators can view all projects.

Project View

The authorised user can access the Project View via the following methods:

  • Creating and submitting a new project (the new project object has been instantiated and validated);
  • From the View Project link after successfully uploading a new project;
  • Clicking the Project Title link on the Listing Projects page.

Figure 11.1 – Project View (Administrator) in a collapsed state

The Project View is displayed in an expanded state by default. Figure 11.1 shows a screenshot of an Administrator User's Project View with the Project Details section collapsed.

On the View Project page, there are two sections:

  • Project details, where the user can View Project Details, and
  • File attachments, where the user can View Attached Files.

Based on the user’s repository and project specific role, different action buttons will be displayed at the bottom of the page.

  • Edit Object takes the user to Edit Project page where the user can edit the project details.
  • Edit Participants button takes the user to the Project Participants screen to view and update participants.
  • Add Child Object button takes the user to an Add Child Object page, through a Dialog to select the type of Child Object and parent-child relationship.
  • Delete Project button allows the authorised user to delete the project.
  • Publish Project button is displayed on the View Project page only when the Project is unpublished and the user is the Principal Investigator.
