Viewing a Child Object


The View Object screen is arrived as a result of Adding a Child object or Editing an Object. It can also be displayed by clicking the object title from any screen where it is displayed as a hyperlink (E.g. View Object, Object Hierarchy).

The View Object screen is displayed as follows (See figure 14.2.1). The following sections are displayed in the View Object page:

  • Parent Details
  • Object Details
  • File Attachment

All the sections are displayed in an expanded state by default. At the bottom of the page, available action buttons are listed for the user.

Figure 14.2.1 – Example screenshot for viewing a Material object & relationships.


Parent Details

The first section of the View Object screen is the Parent Details section. It contains:

  • Parent object URI
  • Parent object Type
  • Parent object Title
  • Parent-Child relationship

Object Details

The Second section is the Object Details section. The title of this section is determined by the object type and the object details too depend on the object type. For the object being viewed, it presents:

  • Available attributes
  • Child objects (if any exist)
  • Refers To objects (if any exist)

File Attachment

The next section is the File attachments section. All the attached file references are listed with their descriptions. The File details can be viewed by clicking the file name link.


Action Buttons

The bottom of the page displays some action buttons, Edit Object, Add Child Object and Delete.  Whether a button is visible or invisible depends on the object’s project Status, the user’s repository role and project specific role, and the object’s position in the domain model as well.  

  • The Edit button takes the user to the Edit Object page for the current object (it is only displayed in Active project, and only available to repository administrator, project administrator and project member). It will be described in next section: Editing a Child Object.
  • The Add Child Object button enables the user to add a child object with the current object as the parent, and will take the user through the Adding a Child Object process as described in the previous section. (This button is only available if the model permits. For example, a Publication object does not allow for any child objects.)
  • The Delete button is only available for the repository administrator or project administrator.
