PODD Prototype Web Service


Login Service

HTTP Method

URL

Parameters

Response

POST

http://<server>/login

username, password

HTTP status code 200 with session information as a cookie



Add Artifact Service

HTTP Method

URL

Parameters

Response

POST

http://<server>/artifact/new

Artifact content in request body. Currently supports RDF/XML only.

The URI of the added artifact.



Get Artifact Service

HTTP Method

URL

Parameters

Response

GET

http://<server>/artifact/[base|inferred]/URI

[base/inferred] – indicates whether the asserted statements or inferred statements should be returned.

URI – identifies the requested artifact

The artifact as stored in PODD is returned.



Edit Artifact Service

HTTP Method

URL

Parameters

Response

POST

http://<server>/artifact/edit/[merge|replace]/URI

RDF statements to be edited are sent in the body of the POST request.

URI – identifies the artifact being modified.

[merge/replace] – indicates whether incoming statements should be merged or they should replace existing statements.

The URI of the modified artifact.
